Same form, two modes. "+ New session" opens it empty (button reads "Create session"); "Edit" opens it filled. Series is just another field here — pick it once, change it if you re-tag. "+ New series…" creates one inline without leaving the form.

Two open calls (need your read)

Duplicate — clones session settings (name +" copy", type, window, series, venue, watch/lobby) and optionally the queued songs, but never participants or votes. New session starts as upcoming. Good for "same setup, next week." → confirm whether songs come along.

Delete — per the audit's delete model: if the session has no votes/participants, delete immediately; if it does, a dialog offers Archive (safe, default) or Cascade delete (type the name to confirm). → confirm you want the full two-path dialog now, or a simple confirm for v1.

Information architecture: Sessions get full CRUD on the card (Open / Edit / Duplicate / Delete) + the shared Create/Edit form. Series already has its own CRUD panel (create / edit / close / leaderboard) — it's now referenced from the session form, not tagged inline. Songs (queue) and Ads (banners) keep their CRUD inside the open session console. Emoji icons here are placeholders — the build uses the app's existing inline-SVG/text glyphs.
