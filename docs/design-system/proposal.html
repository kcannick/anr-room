Proposed unification

One shared vocabulary; per-surface dialects only where earned (overlay)

1 · One token sheet

  • Kill duplicate names: --signal is THE green (drop good/brand-green); --gold: #f5a524 is THE warm accent (drop amber; share cards adopt it); pick ONE purple — --accent: #6d5fe0 — and retire cool or give it a real job.
  • Tokenize the strays: --inset: #0a0814 (inputs, pips, wells); overlay's lighter inks become --ink-dim-video / --ink-faint-video.
  • Ship as one tokens.css imported by all five surfaces.

2 · One radius + spacing scale

  • --r-sm: 10px (chips, small buttons) · --r-md: 12px (buttons, inputs, cells) · --r-lg: 16px (cards) · --r-xl: 18px (page panels) · overlay keeps 22 as its dialect.

3 · One button system

  • .btn + size (sm / md / lg) + intent (primary / ghost / danger / warn).
  • Hierarchy rule: max ONE green primary visible per view — the show-critical action. Saves, utilities, uploads are ghost.

4 · Primitives to extract

  • .chip — one pill primitive (status/badge/perm) with size + intent; distinct muted style for terminal states (completed/archived).
  • .board-row — rank (medal colors everywhere) + name (ellipsis) + mono value. Used by home, overlay, admin, share cards.
  • .data-table — right-aligned mono numeric columns.
  • .modal — one scrim (.82), width scale (sm 340 / md 420 / lg 640), scrim-dismiss unless destructive, gold border = host/system attention.

5 · Keep untouched (it works)

  • DM Sans + Space Mono pairing; the game kit (pips, A/B picks, tier reactions, mono timer); the lift-on-select rule (translateY(-2px) + green fill) — codify it as THE selected state.

6 · Type floor

  • Minimum 12px for any readable text (13px on mobile); eyebrow locked to 11px / 700 / .2em; hints locked to 12px ink-faint.
